# UX設計

## 設計原則

1. **摩擦ゼロ** - 開いたら使える
2. **ゲーム的** - オーバーレイ、ホットキー、常駐
3. **統合表示** - 全部が一つの流れ
4. **欲望肯定** - 目立ちたい、稼ぎたい、それでいい

## ユーザーフロー

### 初回起動

```
ダウンロード
    ↓
起動
    ↓
（裏で鍵自動生成 - ユーザーは知らない）
    ↓
会話が流れている画面が表示
    ↓
使える
```

「アカウント作成」画面は存在しない。
「ログイン」画面も存在しない。
開いたらもう使える。

### 通常利用

```
Gilgaが常駐
    ↓
ゲーム中/作業中に画面端でチャットが流れる
    ↓
気になったらトレイアイコンクリックで展開
    ↓
発言したければ入力
    ↓
またトレイアイコンクリックで最小化
```

## UIモード

### 1. オーバーレイモード（既定）

画面端に常駐する半透明のストリーム。

```
┌──────────────────────────────────┐
│ ゲーム/作業画面                   │
│                                  │
│                                  │
│                    ┌────────────┐│
│                    │ Aさん: xxx ││
│                    │ Bさん: yyy ││
│                    │ Cさん: zzz ││
│                    └────────────┘│
└──────────────────────────────────┘
```

- 位置：画面右下（設定変更可）
- サイズ：幅300px、3〜5行
- 背景：黒、透過60%
- 新着：フェードインで追加

### 2. 展開モード

ホットキーで広げたフル表示。

```
┌────────────────────────────────────┐
│ Gilga                     [−][×] │
├────────────────────────────────────┤
│                                    │
│ Aさん: こんにちは                   │
│ Bさん: 今日も暑いね                 │
│ [投稿] 新作ゲーム買った！ → site.com │
│ Cさん: いいな                      │
│ Dさん: おすすめ教えて               │
│                                    │
├────────────────────────────────────┤
│ [メッセージを入力...]               │
└────────────────────────────────────┘
```

- チャットとSNS投稿が混在
- 投稿には[投稿]ラベル、チャットはラベルなし
- リンクはそのまま表示（宣伝OK）

### 3. 設定モード

最小限の設定。技術用語を隠す。

```
┌────────────────────────────────────┐
│ 設定                              │
├────────────────────────────────────┤
│                                    │
│ 名前: [________]                   │
│                                    │
│ 表示位置: [右下 ▼]                 │
│ 行数: [5 ▼]                        │
│ テーマ: [ダーク ▼]                 │
│                                    │
│ ▶ 詳細設定                         │
│   （リレー、鍵の管理などはここ）     │
│                                    │
└────────────────────────────────────┘
```

## 操作方法

| 操作 | 動作 |
|------|------|
| トレイアイコンクリック | オーバーレイ表示/非表示 |
| Shift+Enter | 送信（入力中） |
| Escape | 閉じる/最小化 |

## 統合ストリーム

### 表示されるもの

```
┌──────────────────────────────────────┐
│ Aさん: 今日のご飯何にする？          │  ← チャット
│ [投稿] Bさん: 新曲リリースしました！  │  ← SNS投稿
│        → https://...                 │
│ Cさん: おめでとう！                  │  ← チャット
│ Dさん: 聴いてくる                    │  ← チャット
│ [投稿] Eさん: セール中！半額です     │  ← SNS投稿（宣伝）
│        → https://shop.example.com    │
└──────────────────────────────────────┘
```

- チャットとSNS投稿を時系列で混合
- リンクは堂々と表示（宣伝を恥じない）
- 区別はつけるが分離はしない

今は全部見せる。過疎を見せない。

## ビジュアル

### カラー

| 要素 | 色 |
|------|-----|
| 背景 | #000000（透過60%） |
| テキスト | #FFFFFF |
| リンク | #4A9EFF |
| 投稿ラベル | #888888 |
| 自分の発言 | #FFD700（ゴールド） |

### ゲーム的要素

- 新着メッセージのフェードイン
- オーバーレイの出現アニメーション
- ホットキーの即時反応
- 最小限のUI、最大限の視認性

### タイポグラフィ

- フォント：システム標準（読みやすさ優先）
- サイズ：14px
- 行間：1.5

## 禁止事項

### 見せないもの

- 「Nostr」という単語
- 「公開鍵」「秘密鍵」
- 「リレー」
- 「NIP」
- 技術的なエラーメッセージ

### 作らないもの

- 投げ銭/Zap機能
- 「サポート」ボタン
- 複雑な設定画面
- チュートリアル（不要なほど簡単に）

## モデレーション

### クライアント側

- ユーザーミュート
- ワードミュート
- 報告機能（将来）

### 思想

- 宣伝はOK（人間の欲望）
- スパムはNG（会話の妨害）
- 線引きはシンプルに
